Data & Methods

Dataset Description

This application integrates three main datasets to provide a complete view of Seattle crime patterns.

1. SPD Crime Data (2008–Present)

Source: Seattle Open Data Portal
Contains report numbers, timestamps, categories, subtypes, and coordinates.

2. Seattle Neighborhood Boundaries

Polygon dataset used to map crimes to neighborhoods.

3. Kaggle Crime Dataset

Used to supplement missing or incomplete SPD entries.

Data Processing

  • Merged datasets using spatial joins
  • Normalized date formats into numeric timestamps
  • Cleaned missing coordinates
  • Standardized category labels

Visualization Design

  • Heatmap for macro-level density analysis
  • Point markers with category colors
  • Popup details and hover interactions
  • Sidebar filters with real-time updates

Key Stats

  • Total crimes: 64,593
  • Property crimes: ~56%
  • Violent crimes: ~5%
  • Top neighborhoods: Capitol Hill, Downtown, Queen Anne

Limitations

  • Reporting bias
  • Missing or inaccurate coordinates
  • Temporal delays in report updates
